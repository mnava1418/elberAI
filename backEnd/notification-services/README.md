# Notification Services

Service responsible for sending all system emails. It is the only service that communicates with Gmail to send emails, and it does so through OAuth2 to authenticate with the sending account.

## What does it do?

This service receives requests from other internal services (primarily auth-services and news_services) and sends emails using Nodemailer with Google OAuth2. Each type of email has its own HTML template.

### Types of emails it sends

**1. Access request** (`POST /email/requestAccess`)
When a user requests access to Elber, this service sends two emails:
- To the **user**: confirmation that their request was received.
- To the **administrator**: a notification with two links (approve / reject). The links take the admin to an auth-service endpoint that processes the decision.

**2. Access response** (`POST /email/accessResponse`)
After the admin approves or rejects:
- If **approved**: the user receives an email with the 6-digit code needed to register.
- If **rejected**: the user receives an email informing them that their request was not approved.

**3. Account verification** (`POST /email/verifyAccount`)
After registration, an email is sent with a Firebase account verification link.

**4. Password recovery** (`POST /email/resetPassword`)
Sends an email with a link to reset the user's password.

**5. Newsletter** (`POST /email/sendNewsletter`)
Sends the newsletter generated by news_services to the list of registered users. This endpoint receives the already-generated HTML content and distributes it.

## Port

Runs on port `4043`.

## Authentication

All endpoints require a valid JWT in the `Authorization: Bearer <token>` header. The secret is configured with the `INTERNAL_TOKEN` variable. Internal services (auth-services, news_services) sign their requests with this secret before calling this service.

## Environment variables

```
NOTIFICATION_PORT=4043
GOOGLE_APPLICATION_CREDENTIALS=  # Path to the JSON file with Google OAuth2 credentials
INTERNAL_TOKEN=                   # JWT secret for validating internal requests
```

## Commands

```bash
npm install
cp .env.template .env
npm run dev     # Development with hot reload
npm run build   # Compile TypeScript
npm start       # Production
```
