# Elber — Mobile App

Elber is the React Native mobile app that allows users to chat with the AI assistant. It is the only point of interaction with the system: from here the user requests access, registers, chats with Elber, and manages their account.

## What does it do?

### Access and registration flow
Elber does not allow open registration. The process is:
1. The user enters their email and requests access.
2. The admin receives an email and manually approves or rejects the request.
3. If approved, the user receives a 6-digit code by email.
4. The user enters that code in the app to validate their access.
5. Only then can they create their account with a name and password.

### Chat with the assistant
The main screen is the chat. The user types or dictates messages by voice, and Elber's responses arrive either as streaming text or as synthesized audio depending on the mode. Each conversation has a title automatically generated by the AI. The chat history is saved and can be resumed at any time.

### Voice input
The user can dictate messages instead of typing them using the device's native speech recognition.

### Voice output (Amazon Polly)
When the user is in voice mode, Elber responds with synthesized speech instead of streaming text. The backend converts the AI response to audio using Amazon Polly (voice: Andrés, es-MX, generative engine) and streams the audio to the app sentence by sentence as base64-encoded MP3 chunks. The app plays each chunk sequentially via a queue as they arrive, creating a natural listening experience. While Elber is talking, a stop button appears so the user can interrupt playback at any time.

### Settings and account
From the settings screen the user can check the status of all backend services (whether they are active or not), manage their profile, and delete their account along with all their data (including the memories Elber has stored about the user).

| Technology | Use |

| React Native 0.80.2 + TypeScript | App framework |
| Firebase Auth | User authentication |
| Socket.io | Real-time response streaming |

| React Navigation | Screen navigation |

| React Native Gifted Chat | Chat interface |
| React Native Voice | Speech recognition (voice input) |
| React Native Sound | Audio playback (voice output) |
| React Native Blob Util | File I/O for caching MP3 chunks |

## Internal architecture

The app follows a layered architecture:

- **`services/`** — All business logic. Components do not call the backend directly; they go through these services.
  - `auth.service.ts` — Access request, code validation, registration, login, logout
  - `chat.service.ts` — Fetch and delete chats
  - `elber.service.ts` — Socket event handling (streaming, complete response, audio chunks, error, title update)
  - `network.service.ts` — HTTP client (Axios) with header and error handling
  - `validation.service.ts` — Form validation (email, password, required fields)
  - `status.service.ts` — Backend service health checks

- **`models/`**
  - `AudioQueue.model.ts` — Singleton that manages sequential MP3 playback. Receives base64 audio chunks, writes them to the device cache, plays them in order via `react-native-sound`, dispatches `elberIsTalking` state updates, and cleans up temp files after playback.

- **`store/`** — Global app state using a Redux pattern (actions / reducers / selectors). Does not use Redux Toolkit. Includes `elberIsTalking` state to drive the voice playback UI.

- **`hooks/`** — Domain-specific hooks: `useChat`, `useVoice`, `useElberStatus`, `useAnimateText`

- **`view/screens/`** — Screens organized in three groups: `auth/` (9 screens for the access and registration flow), `app/` (chat, home, main container), `settings/` (settings, profile, service status)

## Configuration

```bash
cp .env.template .env
```

Required variables:
```
BACK_URL=    # API Gateway URL (HTTP)
SOCKET_URL=  # AI service URL (WebSocket)
```

## Commands

```bash
npm install
cd ios && pod install && cd ..   # iOS only
npm start                        # Metro bundler
npm run ios                      # Run on iOS simulator
npm run android                  # Run on Android emulator
npm run lint                     # ESLint
npm test                         # Jest
```
